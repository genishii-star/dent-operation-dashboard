1. 目的とゴール

  1. Airbnb の ホスト→ゲスト レビュー を自動投稿し、運営工数をゼロにする
  2. 受信レビュー（ゲスト→ホスト）・プライベートフィードバック・★スコアを一元集約
  3. ★低評価レビューへの返信をドラフト生成し、承認フローで投稿
  4. すべてを既存 operation dashboard（index.html のレビュータブ）に統合

期待効果

領域現状導入後
レビュー投稿各アカウント手動・抜け漏れ多発毎日の同期に併走、自動投稿
★スコア把握各アカウントログインして確認ダッシュボードで横断比較
プライベートFB気付かないことが多い物件別に集約・要約
低評価対応気付きが遅い・返信遅れSlack承認→自動投稿

3.3 レビュー本文生成仕様

項目仕様
生成元Claude API（Sonnet推奨、コスト最適化）
言語ゲスト名・国籍・予約言語から自動判定（日 / 英 / 中、デフォルト英）
文字数各言語で2〜4文程度
内容滞在日数・人数・物件名を反映、ブランドトーンに従う
★評価3軸すべて5固定（清潔さ / チェックイン心得 / コミュニケーション）
推薦Yes 固定
プライベートFBPhase 1 では空欄

3.4 安全機構

機構内容
様子見モード初期は「ドラフト生成のみ・実投稿しない」フラグで運用
ホワイトリスト特定物件のみ自動投稿対象 → 段階的に拡大
投稿件数上限1回の同期で最大10件程度
失敗時の継続1件失敗しても次の予約へ進む
重複防止投稿済み予約IDをスプシで管理、再投稿しない

8. セキュリティ・リスク

8.1 セキュリティ要件

  • Airbnbのセッション情報（sessions/*.json）は git にコミット禁止
  • .gitignore に sessions/ を追加
  • macOS Keychain or sops で暗号化保存
  • Claude APIキーは環境変数管理
  • Slack Webhook URL も環境変数管理（過去にコミット履歴あり、再生成要）

8.2 リスクマトリクス

リスク影響対策
高 Airbnb 利用規約違反判定アカウントBAN系統Aは Airhost UI 経由でリスク低、系統Bは週1の低頻度
中 Airhost DOM 変更投稿失敗セレクタ設定ファイル化・Slack通知
中 Airbnb DOM 変更取得失敗同上
中 2FA による自動化失敗ログイン不可初回手動ログイン → セッション長期保持
中 LLM 生成文の品質ばらつき不適切メッセージ様子見期間で完全手動承認
高 低評価返信の誤承認火に油Claude生成 + 人間承認の二重化
